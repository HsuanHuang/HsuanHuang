About

This business big data dashboard is a semester-long project which Cornell collaborated with Axtria, a business big data analytics company. The goal is to build a fully-functioning, visually appealing, and user-friendly new user interface for the data dashboard. In addition, we needed to connect the interface with the back-end data set on Amazon Web Service. There are two designers, one back-end developer, and one data scientist in this team. I designed the user interface and developed the front-end of this data dashboard.Take a look at the prototype!

The problem

Data analysts spends more than 80% of their time on preparing the data, such as parsing, collecting, and joining data. This means that they only has less than 20% of their time on actually analyzing the data and gain insights from it. In addition, data preparation is regarded as the most tedious and boring part of their work.

This is where me and my team want to help with. From the back-end, the data in Amazon Website Service is cleaned and parsed before sending to user's computer. In the front-end, user will have a clean, intuitive, and usable interface where they can easily retrieve data and visualize it. Also, executives and sales can find the answers to basic business questions via interacting with this easy-to-use product.

Define user needs

This product will be used by the business analyst at Axtria. To gain precise user needs, we interviewed a data analyst who currently working in the company. The client has a clear picture in mind of what pages and features they want. The user needs can be put into these major points:

1. Query Page: users can adjust the variables and submit queries to acquire related data
2. Report Page: users can look at the acquire data and play with the variables
3. Data Visualization Page: the acquired data will be visualized here.
4. Login/Logout Page

Start from scratch

The client set a relatively short design phase for this project. They put more attention on the functionality and the development. And would like us to do the same. Therefore, after the client approved the hand-drawn UI created by me, we moved into wireframe phase.

  • Login Page
  • Query Page:
    users can adjust the variables and submit the queries
  • Report Page:
    There are two tables in this page and the users can play with the columns to view the data.
    There is a list of frequently used and history queries on the bottom of the page, so that users don't need to go back to the previous page if they wanted to re-submit the query.
  • Data Visualization Page:
    This part is removed from the scope of this project, so there will only be a dummy data visualization in the final version.

Test it!

Although the client is really satisfied with the wireframe and want us to move on to next design step, we still conduct a quick user-testing and gain some feedbacks. The users can perform without hesitation and error.

1. The users found the top navbar is easy to use and to navigate between pages.
2. The interface looks good and easy to look for target information.

Refine it!

Since we didn't gain significant negative feedback in previous testing, we follow the same design and refine the wireframe into a high-fidelity prototype accordingly. One change we made here is to transform the top navbar as a side menu. The reason why is that the client want to put more pages in the future.

  • Login Page
  • Query Page
  • Report Page
  • Data Visualization Page

Final testing

The client's response to the high-fi interface is very positive and want us to move on to the development phase. Still, we conducted a final testing quickly with 5 Cornell students. The findings are mostly positive and they can complete the user task successfully. One important feedback we got is that the contrast of the interface is high and would easily cause eye fatigue when looking at it for a long time. This finding is important since that the users will look at the interface for a considerable amount of time when working with it. Therefore, we lower the contrast of the interface accordingly.

Front-end Development

Since I worked as the designer and front-end developer in this project, I built the front-end prototype based on the design with the help of Bootstrap. In the mean time, the back-end structure is completed by our data scientist and back-end developer. Therefore, we connect the front-end with the back-end to finish this project and deliver it to the client. The client acceptance is highly positive and think that this product can enhance the working efficiency and effectiveness at Axtria.

My Reflection

In this project, I have the chance to work closely with other designers, back-end developers and data scientists. It enhanced my cross-functional collaboration skill by working with all kind of people. The most important part is to respect their own working style and to make the process visible for everyone in the same time.

Also, I learned how to balance between client's request and users need. The clients want us to move on to the development phase as soon as possible. However, I brought up the importance of iteration to them and made sure that we will still finish the project in expected timeline.
